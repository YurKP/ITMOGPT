# STARS — экосистема студенческих сервисов ИТМО

**STARS** — это набор сервисов, закрывающих разные бытовые и учебные нужды
студента Университета ИТМО: от ответов на вопросы про учёбу и общагу до
интерактивной карты этажей в реальном времени. Все сервисы объединены одной
идеей — **сделать студенческую жизнь в ИТМО удобнее и веселее**.

Репозиторий устроен как **монорепозиторий**: каждый сервис лежит в собственной
папке и работает как самостоятельное Docker-приложение, но сервисы связаны
между собой через общую предметную область и API-интеграцию.

## Что внутри

| Папка | Сервис | Домен | Что делает |
|-------|--------|-------|-----------|
| [`itmogpt/`](itmogpt/) | **ИТМО GPT** | [itmogpt.fun](https://itmogpt.fun) | AI-ассистент студента: чат с RAG по документам ИТМО + три специализированных модуля (Гостомысл, Выготский, Вобщаге) |

## Сервис 1. ИТМО GPT

> Как пройти в библиотеку? Насколько препод лайтовый? Где брать БСК?
> Когда уже в общагу вернут тёплую воду?

**ИТМО GPT** — это AI-помощник, который отвечает на вопросы студентов на основе
реальных документов университета и студенческих чатов. Это не голая LLM, а
**RAG-система**: модель отвечает только тем, что нашла в корпусе знаний, и
честно признаётся, когда данных нет.

### Чем это ценнее обычного чат-бота

ИТМО GPT — это не один чат, а **маршрутизатор к четырём способностям**.
Пользователь пишет запрос одним сообщением, а умный роутер сам определяет,
какой модуль должен ответить:

| Способность | Как вызывается | Что происходит внутри |
|-------------|----------------|------------------------|
| **💬 Чат (RAG)** | Любой вопрос про ИТМО | Поиск по векторной базе документов ИТМО → ответ с учётом истории диалога |
| **📚 Гостомысл** | «найди статьи», «список литературы», «arxiv» | Многоагентный workflow: ArXiv-поиск → ререйтинг → саммаризация → оформление по ГОСТ |
| **🧠 Выготский** | «граф знаний», «путь обучения», «что изучить» | LLM строит граф зависимостей тем → оценивает знания пользователя → считает персональный путь обучения |
| **🏠 Вобщаге** | «общага», «этаж», «соседи» | Живой запрос к API Вобщаге + RAG-контекст |

### Архитектура

```
                ┌─────────────── ИТМО GPT ───────────────┐
                │                                         │
Пользователь ─► │  FastAPI (web) ──► ITMORouter          │
   (сайт /     │       ▲                 │               │
    Telegram)  │       │                 ├──► RAG-chain  │ ─► ChromaDB
                │       │                 ├──► Гостомысл  │ ─► ArXiv + агенты
                │  WebSocket ◄────────────┤──► Выготский  │ ─► LLM-граф
                │  (progress-стрим)       └──► Вобщаге    │ ─► REST в vopstchage
                │                                         │
                └─────────────────────────────────────────┘
```

**Ключевые компоненты:**

- **FastAPI-бэкенд** (`app/web/api.py`) — HTTP-эндпоинты (`/api/chat`,
  `/api/vygotsky/*`) и WebSocket `/ws/gostomysl` для потоковой передачи
  прогресса работы агентов.
- **ITMORouter** (`app/modules/router.py`) — мозг системы. По ключевым словам
  маршрутизирует запрос в нужный модуль; при запросе про общагу подмешивает
  живые данные из Вобщаге в RAG-контекст.
- **RAG-цепочка** (`app/rag/`) — `RecursiveCharacterTextSplitter` →
  векторные эмбеддинги → **ChromaDB** как локальное векторное хранилище →
  `RunnableWithMessageHistory` для памяти диалога.
- **Гостомысл** (`app/modules/gostomysl/`) — конвейер из пяти агентов:
  `QueryAgent` (расширение запроса) → `SearchAgent` (ArXiv) →
  `RankingAgent` (многоэтапный ререйтинг) → `SummaryAgent` (саммари) →
  `GOSTFormatter` (библиография по ГОСТ).
- **Выготский** (`app/modules/vygotsky/graph.py`) — LLM генерирует граф
  знаний в JSON (`nodes` + `edges` зависимостей), затем оценивает, какие
  топики уже знает пользователь, и считает топологический путь обучения.
- **Telegram-бот** (`app/bot/telegram.py`) — второй клиентский интерфейс,
  long-polling. Делит с вебом ту же RAG-инфраструктуру.

**Корпус знаний** (`itmogpt/data/`) собран из:
- официальных сайтов ИТМО (`itmo.ru`, `student.itmo.ru`, `lib.itmo.ru`, …),
- страниц кафедр и персоналий,
- экспортированных диалогов из Telegram-чатов ИТМО (парсер
  `scripts/parse_telegram_chats.py`).

### Технологии

Python 3.12 · FastAPI · LangChain · LangGraph · ChromaDB · sentence-transformers ·
python-telegram-bot · Docker · Caddy.

## Сервис 2. Вобщаге

> Где сейчас тусуется больше всего народу? На каком этаже громче?
> А что в комнате 1203 — там живёт Кот?

**Вобщаге** — это **многопользовательская карта общежития в реальном времени**.
Студенты заходят на сайт, выбирают аватар и скин, и оказываются на карте
своего этажа — как в старых добрых 2D-играх. Можно перемещаться, заходить в
комнаты, расставлять эмодзи-предметы и взаимодействовать с окружением. Все
действия видны остальным игрокам мгновенно.

### Чем это ценнее обычного сайта

| Возможность | Как реализовано |
|-------------|----------------|
| **Реальное время** | Socket.IO с WebSocket-транспортом: движения и события летят всем участникам без перезагрузки страницы |
| **Persistent-сессии** | Каждый игрок получает токен с TTL 48 ч в **Redis**; «умершие» аккаунты автоматически вычищаются |
| **Игровая логика** | На карте живут NPC: Кот 🐈 на холодильнике и Бессонный Дипломник 🧟‍♂️ как босс `dorm_stone` |
| **Аналитика** | Все события (`login`, `connect`, `join_room`, `interact`, `disconnect`) пишутся в JSONL-лог для последующего анализа через `jq` |
| **AI-репорты** | Эндпоинты `/api/admin/report`, `/api/newspaper`, `/api/dorm-summary` формируют саммари событий и «общажную газету» |
| **Интеграция с ИТМО GPT** | `/api/dorm-summary` отдаёт агрегированные данные (онлайн по этажам, топ комнат, эмодзи) — ИТМО GPT подтягивает их в RAG-контекст при вопросах про общагу |

### Архитектура

```
Браузер ──► Nginx (frontend) ──► статика React
              │
              ├──► /          → React SPA (Vite-сборка)
              └──► /socket.io/ ─► FastAPI (backend) ──► Redis (state + pub/sub)
                                      │
                                      └──► JSONL-лог (аналитика)

Caddy ──► порты 80/443 ──► TLS (Let's Encrypt) ──► frontend:80
```

**Ключевые компоненты:**

- **Frontend** — React + Vite + Tailwind. Раздаётся через **Nginx**, который
  также проксирует `/socket.io/` на бэкенд.
- **Backend** (`backend/app/`) — FastAPI + `python-socketio`:
  - `state_manager.py` — вся игровая логика и работа с Redis,
  - `analytics.py` — JSONL-логгер событий,
  - `ai_reports.py` — генерация саммари и газет,
  - `main.py` — REST + Socket.IO сервер.
- **Redis** — единый источник правды для позиций игроков, предметов и сессий.
- **Caddy** — входная точка с автоматическим HTTPS через Let's Encrypt.

### Технологии

Python 3.11 · FastAPI · python-socketio · Redis · React · Vite · Tailwind ·
Nginx · Docker · Caddy.

## Как сервисы связаны

Главная фишка экосистемы — **сервисы не изолированы, а обмениваются данными**:

```
┌─────────────────┐   REST /api/dorm-summary   ┌─────────────────┐
│   ИТМО GPT      │ ◄───────────────────────── │    Вобщаге      │
│  (itmogpt.fun)  │      живые данные общаги    │ (вобщаге.fun)   │
└─────────────────┘                             └─────────────────┘
        │                                                │
        └──────────► единое студентское окружение ◄──────┘
```

Когда студент спрашивает у ИТМО GPT «что сейчас происходит в общаге?», тот
не выдумывает, а идёт реальным HTTP-запросом в Вобщаге, забирает текущую
статистику по этажам и комнатам, и формирует ответ на живых данных. Так
бэкенды становятся частями одной экосистемы, а не двумя независимыми сайтиками.

## Инфраструктура и развёртывание

Оба сервиса следуют одному и тому же подходу — **stateless-приложение в
Docker + обратный прокси с автосертификатами**:

- **Хостинг:** Yandex Cloud (Compute Cloud + VPC), Ubuntu 22.04/24.04.
- **Контейнеризация:** Docker + Docker Compose; каждый сервис описан
  собственным `docker-compose.yml`.
- **HTTPS:** Caddy во обоих сервисах автоматически получает и продлевает
  сертификаты Let's Encrypt по домену — ручное управление TLS не требуется.
- **LLM-провайдер:** YandexGPT (`yandexgpt/latest`, `text-search-doc/latest`)
  — нативно доступен из инфраструктуры Yandex Cloud, без сетевых блокировок.
  Архитектура LLM-слоя абстрактна: ИТМО GPT одинаково поддерживает и
  YandexGPT, и любой OpenAI-совместимый API.

Требования к окружению и переменные приведены в `.env.example` внутри каждой
папки.
